# 海辰BMS2.0上位机使用手册 — HMI 行为要点

> **数据来源**：[`海辰BMS2.0上位机使用手册(内部)-V2.8.pdf`](../../../docs/海辰提供的文件/产品规格书/海辰BMS2.0上位机使用手册(内部)-V2.8.pdf)  
> **用途**：定义上位机 HMI 的预期行为，指导 `rbms_tcp_sim` 模拟器与 HMI 的交互逻辑。

---

## 1. 通讯架构

### 1.1 以太网连接 — 客户端模式（最常用）

```
┌────────────┐    TCP :5001    ┌────────────┐    cmdGroup=0x03    ┌────────┐
│   HMI 上位机 │ ◄──────────────► │ 三级板 A 核  │ ◄────────────────► │ 二级板  │
│  TCP Server │                 │ (BBMS A)   │                    │ (RBMS) │
└────────────┘                  └────────────┘                    └────────┘
```

- HMI 作为 **TCP Server**，监听端口 **5001**
- 三级板 A 核作为 **TCP Client** 主动连接 HMI
- 默认 IP：`192.168.1.136:5001`
- HMI 与二级板不直连（以太网模式下），所有数据经三级板 A 核转发

### 1.2 以太网连接 — 服务端模式（直连二级板）

```
┌────────────┐    TCP :5001    ┌────────┐
│   HMI 上位机 │ ◄──────────────► │ 二级板  │
│  TCP Server │                 │ (RBMS) │
└────────────┘                  └────────┘
```

- 无三级板时使用
- HMI 作为 TCP Server，二级板作为 Client 连接
- 最多支持 **12 块二级板**
- 需关闭电脑防火墙

### 1.3 CAN 连接

- 通过 TSMaster CAN 盒，以 CANFD 帧通讯
- 支持同星 CAN 盒系列

## 2. 关键通讯参数

| 参数 | 默认值 |
| :--- | :---: |
| HMI 服务端口 | **5001** |
| 默认 IP | 192.168.1.136 |
| 三级板 A 核 IP 偏移规则 | 电脑 IP = 三级板 IP 第 4 段 + 13 |
| 二级板 IP 规则 | 写入 IP 第四段 + Rack 编号 |

## 3. 用户权限等级

| 等级 | 用户名 | 密码 | 功能范围 | 对模拟器的影响 |
| :--- | :--- | :--- | :--- | :--- |
| 用户 | User | 无 | 只读：查看状态、故障、版本 | 模拟器无需处理写操作 |
| 售后 | AF | 需申请 | 部分写：IO 控制、程序升级 | 需要处理 CtlWord / DO 控制 |
| 研发 | Root | 需申请 | 全部：参数设置、标定、XCP 等 | 需要处理所有读写命令 |

## 4. 状态监控行为

### 4.1 系统状态

- 连接成功 **无提示**，失败则弹框提示
- Bank 汇总信息 + Rack 汇总信息
- **电流**：充电为负值，放电为正值
- **无效值**：
  - 电流无效值：`-2000A`
  - 温度无效值：`-40°C`
- **通讯状态**：
  - 绿色灯 = 通讯正常
  - 黄色灯 = 通讯异常

### 4.2 故障告警

- 支持按 BankA / BankM / Rack 分层查看
- Rack 故障：所有在线簇的对应故障位做 **或逻辑**
- **灰色字体** = 该故障未使能
- **白色字体** = 故障已使能
- **红色圆灯** = 当前发生故障
- **绿色圆灯** = 无故障
- 故障使能/不使能可通过页面配置，写入板端
- 按故障等级分区显示，列排序

### 4.3 详细数据

- 电芯电压格式：`1#Cell1` ~ `8#Cell40`（8 Pack × 40 Cell）
- 均衡状态：1=均衡中，0=未均衡
- 温度探头格式：`1#Probe1` ~ `8#Probe10`
- 自放电率：电芯的月自放电差异

### 4.4 TMS 信息

- TMS 状态：热管理汇总信息
- AllBank / AllRack 选择时会清空
- 温度控制阈值
- 工作模式与温度设定

## 5. 控制与标定行为

### 5.1 IO 控制

| 层级 | 操作 | 行为 |
| :--- | :--- | :--- |
| Bank | 本地 | 所有 Rack 下高压（主正/主负断开），切本地模式，使能=禁止，连接=断开 |
| Bank | 远程 | 无一级故障时自动上高压，使能=使能，连接=连接 |
| Rack | 本地 | 断开时成功切换，否则提示成功但实际可能失败；使能=禁止，连接=断开 |
| Rack | 远程 | 无条件切换；使能=使能，连接=连接 |

- 故障清除与绝缘检测：本地/远程均可实现
- 参数标定：填入数值 → 设置 → 提示成功
- 保存 NVM：保存累计容量/能量/时间/SOC/SOH 快照到本地文件

### 5.2 参数配置

- 回读：读取当前系统各模块配置值
- 修改：解锁 → 填写修改值 → 写入 → 弹窗提示
- 阈值修改后需回读确认
- NVM 参数清除：清除后需重新上下电生效

---

## 6. 版本管理

- 控制器 ID 写入（手动编址）：针对单个 Rack
- 时间写入：年月日时分秒写入板端
- IP 设置：三级板（ETH0=HMI 通讯，ETH1/ETH2=EMS 通讯）；二级板（客户端连接服务端）
- 二级板实际 IP = 写入 IP 第四段 + Rack 编号

---

## 7. 程序升级行为

### 7.1 Rack 程序刷写

- 支持多板同时升级（仅二级板）
- 支持以太网（需三级板）和 CAN 盒（无需三级板）
- 升级流程：0%-2% 准备 → 3%-98% 传输 → 99%-100% 校验 + 重启

### 7.2 Bank 程序刷写

- 浏览器跳转到 SWU 升级页面
- 三级板双系统升级

### 7.3 Rack 参数刷写

- 选择固件 + 选择参数文件（`_RBMS` 后缀）
- 仅修改 GlobalDD 全表 + SystemConfiguration Import 参数

---

## 8. 数据管理行为

- 通讯建立后自动存储 CSV 文件
- 存储路径：`\HithiumBESS_HRD\`
- 按连接时间命名目录（如 `20250406142017`）
- Rack 数据：N 个簇对应 N 个文件
- SSD 数据下载：选择日期 → 下载到 `SSD_Data/` 目录

---

## 9. 对模拟器的关键约束

| 序号 | 要点 | 模拟器需满足 |
| :---: | :--- | :--- |
| 1 | HMI 为 TCP Server :5001 | HMI 通道需作为 TCP Client 连接 |
| 2 | 连接成功无提示 | 模拟器需持续上送状态数据 |
| 3 | 电流无效值 = -2000A | CSV 默认值需体现 |
| 4 | 温度无效值 = -40°C | CSV 默认值需体现 |
| 5 | 通讯状态绿/黄判定 | 模拟器需保持心跳（SumInfo 1s）不掉线 |
| 6 | Rack 故障 = 所有在线簇或逻辑 | 模拟器目前仅单簇，故障位图需正确 |
| 7 | 均衡状态 1=均衡中 0=未均衡 | CellBalSt 报文需符合 |
| 8 | 二级板 IP = 写入IP + Rack编号 | 模拟器可忽略（仅本地模拟） |
| 9 | 参数写入需应答 state | CtlWord 应答 1B state=0 |
| 10 | 多板升级同时进行 | 不属于模拟器 MVP 范围 |
