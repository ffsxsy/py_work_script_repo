# BMS2.0 数据包格式 V2

> **数据来源**：[`BMS2.0底软通信协议.pdf`](../../../docs/海辰提供的文件/开发资料/BMS2.0底软通信协议.pdf) — 数据包格式V2  
> **代码映射**：[`src/rbms_tcp_sim/protocol.py`](../src/rbms_tcp_sim/protocol.py)

## 字节序

多字节数据：**低字节在前，高字节在后**（小端格式，Little-Endian）。

## V1 vs V2 差异

| 字段 | V1 | V2 |
| :--- | :--- | :--- |
| 版本 (version) | 1 | 2 |
| 网络地址 | `src`(1B) + `dest`(1B) | `src`(1B) + `srcSub`(1B) + `dest`(1B) + `destSub`(1B) |
| Body 头长度 | 6 字节 | **8 字节** |

V2 新增了 **srcSub / destSub** 子地址字段，以支持多设备（多簇、多核）寻址。

## V2 帧结构

```
Byte1  Byte2-3    Byte4-5  Byte6   Byte7   Byte8   Byte9   Byte10  Byte11  Byte12  Byte13  Byte14-n
┌─────┬──────────┬────────┬──────┬───────┬───────┬───────┬───────┬───────┬───────┬───────┬──────────┐
│Head │Version&  │CRC16   │ src  │srcSub │ dest  │destSub│trans- │frameId│cmdGrp │cmdId  │  data    │
│0xA5 │ datalen  │        │      │       │       │       │portTyp│       │       │       │          │
└─────┴──────────┴────────┴──────┴───────┴───────┴───────┴───────┴───────┴───────┴───────┴──────────┘
│←── LinkMsg (5B) ──→│←── Body (8B + data) ─────────────────────────────────────────────────────→│
```

### LinkMsg（5 字节）

| 偏移 | 字段 | 长度 | 说明 |
| :---: | :--- | :---: | :--- |
| 0 | `head` | 1B | 协议头部，固定 `0xA5` |
| 1-2 | `version : 5` | 5 bit | 协议版本，当前 V2 |
| 1-2 | `len : 11` | 11 bit | data 总长度（byte6 起的长度），最大值 1500 |
| 3-4 | `check` | 2B | byte6~n 的 **CRC16-Modbus** 校验结果 |

```c
struct __attribute__((packed)) LinkMsg {
    uint8_t head;              // 0xA5
    uint16_t version : 5;      // 当前为 V2
    uint16_t len : 11;         // Body 总长，max 1500
    uint16_t check;            // Body CRC16-Modbus
};
```

### Body 头（8 字节）

#### NetworkMsgV2（4 字节）

| 偏移 | 字段 | 长度 | 说明 |
| :---: | :--- | :---: | :--- |
| 0 | `src` | 1B | 源器件地址 |
| 1 | `srcSub` | 1B | 源器件子地址 |
| 2 | `dest` | 1B | 目的器件地址 |
| 3 | `destSub` | 1B | 目的器件子地址 |

```c
struct __attribute__((packed)) NetworkMsgV2 {
    uint8_t src;
    uint8_t srcSub;
    uint8_t dest;
    uint8_t destSub;
};
```

#### TransportMsgV1（2 字节）

| 偏移 | 字段 | 长度 | 说明 |
| :---: | :--- | :---: | :--- |
| 4 | `transportType` | 1B | 报文类型 |
| 5 | `frameId` | 1B | 帧序号，每个主动发出的帧 +1 |

```c
struct __attribute__((packed)) TransportMsgV1 {
    uint8_t transportType;     // 0x01: 不需要回应; 0x02: 需要回应; 0x03: 回应报文
    uint8_t frameId;           // 帧序号递增
};
```

#### ExpressionMsgV1（2 字节 + data）

| 偏移 | 字段 | 长度 | 说明 |
| :---: | :--- | :---: | :--- |
| 6 | `cmdGroup` | 1B | 命令组 |
| 7 | `cmdId` | 1B | 命令号 |
| 8+ | `data[]` | 变长 | 消息 payload |

```c
struct __attribute__((packed)) ExpressionMsgV1 {
    uint8_t cmdGroup;
    uint8_t cmdId;
    uint8_t data[];
};
```

## 传输类型（transportType）

| 值 | 含义 |
| :---: | :--- |
| `0x01` | **不需要应答** — 周期上送报文（SumInfo、Volt、Temp 等） |
| `0x02` | **需要应答** — 控制指令（CtlWord 等），接收方需回 `0x03` |
| `0x03` | **应答报文** — 对 `0x02` 的回复，同 frameId |

## 设备节点地址

| 设备 | `dev` | `sub` | 说明 |
| :--- | :---: | :---: | :--- |
| 广播 | `0x00` | 0 | 所有设备 |
| 上位机与A核组播 | `0x01` | 0 | — |
| 上位机（电脑） | `0x01` | 1 | HMI |
| 三级板 A 核 | `0x01` | 2 | BBMS A-core |
| 三级板 A 核 OTA | `0x01` | 3 | OTA 进程 |
| HIL 工具 | `0x01` | 4 | 硬件在环 |
| 自动化工具 | `0x01` | 5 | 自动测试 |
| 三级板 M 核 | `0x03` | 1 | BBMS M-core |
| 二级板 | `0x04` | N | RBMS，N 为 Rack ID |
| 主动均衡板 | `0x05` | N | 主动均衡 |

## CRC16-Modbus 算法

校验范围：Body 所有字节（byte6 ~ byteN）。  
算法：标准 Modbus CRC-16（多项式 `0x8005`，初始 `0xFFFF`）。

```python
def crc16_modbus(data: bytes) -> int:
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 0x0001:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc
```

## 帧长计算

```text
总帧长 = 5 (LinkMsg) + 8 (Body 头) + len(data)
       = 13 + len(data)
```

其中 `LinkMsg.len = 8 + len(data)`（byte6 起的长度）。

## 数据包格式 V2 变更

| 版本 | 变更 |
| :--- | :--- |
| V1 | src(1B) + dest(1B)，无子地址；Body 头 6B |
| V2 | 新增 srcSub + destSub；Body 头 8B；version 字段设为 2 |
