> **文档定位**：定义 `rbms_tcp_sim` 的功能需求与非功能需求，作为开发、测试、验收的统一依据。  
> **目标读者**：产品经理、测试工程师、联调工程师、后续维护开发者。  
> **版本**：V1.0 | **最后更新**：2026-06-10

## 1. 引言

### 1.1 背景

在 BMS 堆控系统中，BBMS（三级板）通过 TCP 与各 RBMS（二级板，每簇一块）通信，采集电池簇的电压、温度、故障等信息并下发控制指令。  
`rbms_tcp_sim` 是一个纯 Python 实现的 **RBMS 模拟器**，在开发和联调阶段替代真实 RBMS，使得 BBMS 固件和 HMI 上位机可以在无硬件环境下验证功能。

### 1.2 范围

本文档涵盖 `rbms_tcp_sim` 的全部必须功能，包括：

- 双通道 TCP 通信（HMI 通道 + BBMS 通道）
- 六类周期上送报文
- 下行命令接收与应答
- CSV 驱动的数据源与热加载
- 运行时行为（心跳、animate 缓变、双通道隔离）

## 2. 系统概述

### 2.1 模拟器角色

模拟器扮演 **第 1 簇 RBMS（二级板）**，同时维护两条独立 TCP 通道：

```mermaid
flowchart LR
    HMI[HMI 上位机<br/>TCP Server :5001]
    SIM[RBMS Simulator<br/>dev=0x04 sub=1]
    BBMS[BBMS 主控<br/>TCP Client]

    SIM -- TCP Client --> HMI
    BBMS -- TCP Client --> SIM
```

### 2.2 约束

- 一次进程固定模拟 **第 1 簇**（`rack_id=1`，协议 `src_sub=1`）
- 不模拟多簇并发
- 不实现 Matrix 全部 35 条消息，仅实现 MVP 范围内的 Tx 6 条 + Rx 2 条

## 3. 功能需求

### 3.1 通信（FR-COM）

| ID | 需求 | 验证方式 |
| :-- | :--- | :------- |
| FR-COM-01 | 模拟器 HMI 通道作为 **TCP Client**，主动连接 HMI 上位机（默认 `127.0.0.1:5001`） | 抓包确认 SYN |
| FR-COM-02 | HMI 断线后自动重连，重连间隔可配置（默认 5s） | kill 连接后观察重连日志 |
| FR-COM-03 | BBMS 通道作为 **TCP Server**，监听配置地址（默认 `0.0.0.0:5002`） | `netstat` 确认 LISTEN |
| FR-COM-04 | BBMS 通道同一时间只接受 **1 个** BBMS 连接；新连接踢旧连接 | 连两个 BBMS 观察日志 |
| FR-COM-05 | HMI 通道与 BBMS 通道**同时就绪、并行运行**，无先后依赖 | 启动后两通道日志确认 |
| FR-COM-06 | 通信使用 BMS 2.0 私有协议 V2：[R5] §V2 帧结构 | — |
| FR-COM-07 | 每帧必须包含 5B LinkMsg（head=0xA5, version=2, CRC16-Modbus） | 测试 `test_protocol.py` |
| FR-COM-08 | 多字节字段采用**小端**编码 | — |
| FR-COM-09 | 所有 Tx 帧 `transportType=0x01`（不需要应答） | — |

### 3.2 HMI 通道（FR-HMI）

| ID | 需求 | 验证方式 |
| :-- | :--- | :------- |
| FR-HMI-01 | HMI 通道 Tx 目标地址 `dest=0x01:0x00`（上位机与A核组播） | 抓包确认 dest |
| FR-HMI-02 | HMI 通道 Rx 来源地址不限制，处理 CtlWord 和 SafetySignal | 测试 |
| FR-HMI-03 | HMI 通道以 `src=0x04:1`（DEV_RBMS:rack_id）作为源地址 | — |

### 3.3 BBMS 通道（FR-BBMS）

### 3.5 报文内容（FR-MSG）

| ID | 需求 | 验证方式 |
| :-- | :--- | :------- |
| FR-MSG-01 | SumInfo 的 `RBMS_StrCtrlHb`（payload byte 159-160）每周期 **自增 1**，覆盖 CSV 值 | 抓包确认 +1 |
| FR-MSG-02 | SumInfo 的 `RBMS_St`（payload byte 0）bit0=Enable=1, bit1=Online=1 | — |
| FR-MSG-03 | 其余 payload 字段由对应 CSV 点表指定初始物理值，按 `raw = round((物理值 - offset) / resolution)` 编码 | 测试 `test_matrix_messages.py` |

| FR-MSG-05 | CSV 中标记 `animate=false` 的信号，物理值保持不变 | — |
| FR-MSG-06 | 发生故障的位（Fault 报文）对应 bit=1 | 测试 |

## 4. 非功能需求

| ID | 需求 | 指标 |
| :-- | :--- | :--- |
| NFR-01 | **协议合规**：所有帧必须通过 `parse_check_frame()` 校验（head=0xA5, version=2, CRC16 正确） | 100% |
| NFR-02 | **代码质量**：通过 `ruff check`、`ruff format --check`、`ty check` | 无错误 |
| NFR-03 | **测试覆盖率**：核心逻辑（protocol、handlers、matrix runtime）单元测试覆盖 | ≥80% |
| NFR-04 | **CPU 占用**：空闲时（仅周期上送）CPU 使用率 | <5% 单核 |
| NFR-05 | **内存占用**：稳态内存 | <50MB |
| NFR-06 | **热加载延迟**：CSV 修改 → 下一周期生效 | ≤1 tick |
| NFR-07 | **断线重连**：HMI 断线后在 `reconnect_interval_s` 内自动重连 | 100% |
| NFR-08 | **Python 版本**：Python 3.13 | — |

## 5. 接口定义

### 5.1 CLI 接口

```bash
rbms-sim [OPTIONS]

选项：
  --config PATH               TOML 配置文件路径（默认 config/rbms_sim.toml）
  --init-config               生成默认 TOML 后退出
  --init-matrix-config        生成六类 CSV 模板后退出
  --interval FLOAT            基准 tick（秒）
  --no-reply                  关闭 CtlWord 自动应答
  -v, --verbose               DEBUG 日志
```

### 5.2 TCP 接口

| 通道 | 角色 | 地址 |
| :--- | :--- | :--- |
| HMI 通道 | TCP Client → Server | 默认 `127.0.0.1:5001` |
| BBMS 通道 | TCP Server ← Client | 默认 `0.0.0.0:5002` |

### 5.3 设备地址

| 实体 | dev:sub |
| :--- | :--- |
| RBMS 模拟器（HMI Tx 源） | 0x04:01 |
| RBMS 模拟器（BBMS Tx 源） | 0x04:01 |
| HMI 通道 Tx 目标 | 0x01:00 |
| BBMS 通道 Tx 目标 | 0x03:01 |

## 6. 验收准则

### 6.1 必须通过

1. **`uv run pytest`** — 45 项测试全部通过
2. **`uv run ruff check src tests`** + **`uv run ty check src tests`** — 无错误
3. **`uv run rbms-sim`** 启动后日志显示 `已连接` + `Server 监听`

### 6.2 协议验证

| 验收项 | 方法 |
| :----- | :--- |
| 帧头 0xA5, version=2 | Wireshark Lua dissector 或抓包 |
| CRC16-Modbus 对 body 校验正确 | 修改 payload 后应解帧失败 |
| 六类报文按周期准时发送 | tcpdump 30s 窗口 |
| HMI 断线重连 | kill 连接后 5s 内重连 |
| CtlWord 应答 1B state=0 | 发送 CtlWord 后捕获回复帧 |
| 双通道独立 frameId | 两通道 frameId 序列无交叉 |

| 需求 | 原因 |

## 8. 术语表

| 术语 | 说明 |
| :--- | :--- |
| BBMS | Bank BMS（三级板），BMS 堆控主控制器 |
| RBMS | Rack BMS（二级板），每簇一块，采集电池数据 |
| HMI | 上位机，PC 端监控软件 |
| LinkMsg | 5 字节链路层头（0xA5 + version&len + CRC16） |
| Matrix | LAN Matrix 协议规范，定义全部消息 |
| SumInfo | 概要信息报文 |
| StrCtrlHb | 心跳计数器（SumInfo 内字段） |
| animate | CSV 信号缓变模式，按正弦波更新物理值 |
| MVP | Minimum Viable Product，当前实现范围 |
| payload_len | 报文数据体长度（不含 LinkMsg 和 Body 头） |
